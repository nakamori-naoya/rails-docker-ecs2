【ポートフォリオのCUD】
`ポートフォリオの投稿`
```
【基本コース】
システムは、「ポートフォリオ投稿」ページを表示する。
顧客は、ポートフォリオのタイトル・ポートフォリオのURL・画像(1枚以上)・GithubのURL・説明文をフォームに入力し、カテゴリ検索欄からカテゴリを検索し、１つ以上選択の上で送信ボタンをクリックする。
システムは、説明文の長さが長すぎないか検証する。そして、データベースにデータを保存する。
その後、システムは、「ポートフォリオ詳細」ページを表示する。

【代替コース】
<説明文の長さが長すぎる場合>
システムは、「説明文が長すぎます。○○字以内で入力して下さい」メッセージを表示する。

<すでに同じタイトルのポートフォリオが存在する場合>
システムは、「すでに同じタイトルのポートフォリオが存在するので、ポートフォリオの名前を変更して下さい」メッセージを表示する。

<顧客が必須項目の入力・選択を行っていない場合>
システムは、顧客に「送信」ボタンを押させないようにする。

<選択した画像を削除したい場合>
顧客は、画像をクリックする。
システムは、「本当に削除しますか？」というアラートを出し、顧客が「はい」をクリックした場合、システムは画像を削除する。

<追加したカテゴリを削除したい場合>
顧客は、削除したい「カテゴリ名」ボタンをクリックする。
システムは、当該「カテゴリ名」を削除する。

<新規でカテゴリを追加する場合>
顧客は、検索した「カテゴリ名」の横に「登録」ボタンがあるので、それをクリックする。


<新規で追加するカテゴリが既に存在する場合>
システムは、重複したカテゴリを排除し、ユーザーに「そのカテゴリは既に作成されています」と表示する。

```


`ポートフォリオの編集`
```
【基本コース】
顧客は、「顧客プロフィール詳細ページ」の編集したいポートフォリオの「ポートフォリオカード」内にある「編集」ボタンをクリックする。
システムは、データベースから当該ポートフォリオの情報を取得し、「ポートフォリオ編集」ページを表示する。
顧客は、ポートフォリオの編集を行い、「送信」ボタンをクリックする。
システムは、説明文の長さが長すぎないか検証する。そして、データベースのデータを更新する。
その後、システムは、「ポートフォリオ詳細」ページを表示する。

【代替コース】
『ポートフォリオの投稿』のものと同様
```


`ポートフォリオの削除`
```
【基本コース】
顧客は、「顧客プロフィール詳細ページ」の削除したいポートフォリオの「ポートフォリオカード」内にある「削除」ボタンをクリックする。
システムは、顧客に「本当に削除しますか？」というアラート画面を表示する。
顧客が、アラートに対して「はい」を選択した場合、システムは、当該ポートフォリオを削除する。

【代替コース】
<顧客がアラートに対して「いいえ」を選択した場合>
システムは、アラート画面を消す。
```
